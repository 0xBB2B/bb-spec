# 报告模板

本文件供 SKILL.md §3.4（机械层自动修回显）与 §6（语义层报告）使用。

---

## 1. 机械层自动修复回显（SKILL.md §3.4）

主对话批量修完所有满足"直接修复条件"的机械问题后，一次性回显（不要逐条问用户）：

```
[机械层] 自动修复 N 个：
- [已修] file:line · 类型 · 一句话说明
- [已修] file:line · 类型 · 一句话说明
- ...

机械层待决策 M 个（与语义审查报告一起输出）
```

如果 M = 0，省略最后一行。

如果 N = 0，行首改为 `[机械层] 无可自动修复项`。

---

## 2. 概览（SKILL.md §6.1）

语义审查完成后输出，不列详细问题清单：

```
文档审查完成

范围：N 个文档（主 X / 关联 Y）
机械层：自动修复 A 个；待决策 B 个
语义层：跑了 K 个 agent（Claude / Codex）
（Codex 不可用时：本次缺失 Codex 跨模型那一份）

合并去重后共 M 条待决策：
- 🔴 BLOCKER：a 条（其中 ⭐ 交叉验证 a' 条）
- 🟡 IMPORTANT：b 条（其中 ⭐ 交叉验证 b' 条）
- 🟢 NIT：c 条

交叉验证强信号（独立 agent 同时指出）：
- <项 1 简述>
- <项 2 简述>
```

---

## 3. 详细报告（SKILL.md §6.2）

按严重度分组（🔴 → 🟡 → 🟢），每条按以下格式：

```
### [🔴/🟡/🟢] <项 N> · <类型> · <one-line 标题>  [⭐ 交叉验证（可选）]

**位置**：`file:lines`（多处则全部列出）
**来源**：[机械待决策] / Agent 1 (Claude) / Agent 2 (Codex) / ...
**事实**：
<3-5 行说明：位置 A 说 X，位置 B 说 Y / 或具体冲突点>
**影响**：
<读者会误解什么 / 开发者会做错什么决策；落到具体一项>
**建议**：
<修法摘要，≤ 3 行；不引用原文片段、不做修改前后对比>
```

**类型枚举**：
- 机械待决策：断链 / 锚点失效 / 引用未定义 / 术语不一致 / 标题层级 / 表格列数
- 语义：前后矛盾 / 单一真理来源 / 承诺与实际 / 历史包袱

**写作约束**：
- 用大白话说清问题本质，不引用原文片段，不做修改前后对比
- 避免"可能会"、"容易导致"等模糊说法
- 历史包袱类必须说明为何对**当前读者**已无价值

---

## 4. 处理建议表（SKILL.md §6.3）

```
建议处理顺序：

| 优先 | 项 | 处理方式 |
| --- | --- | --- |
| 立即 | <项 N>, <项 M> | 主对话单独触发修复 |
| 跟进 | <项 K> | 列 followup issue |
| RA | <项 L> | 在文档前言加 Risk Accept 说明 |

要修哪些？（回复项编号 / "全部 BLOCKER" / "到此结束"）
```

跨文件影响（机械层修复时已涉及 / 语义建议如修复会涉及）单独提示：

```
跨文件影响（如选择修复，需同步处理）：
- <影响的代码 / Proto / 配置文件路径与改动点>
```
